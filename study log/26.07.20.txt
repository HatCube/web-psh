빌드툴
gradle : groovy 기반의 의존성 관리 툴
ㄴ build.gradle
maven : XML 기반의 빌드 및 의존성 관리 도구
ㄴ pom.xml

프로젝트의 기본적인 구조
레이어드 아키텍쳐 : 계층으로 나눈다.
표현계층 : Controller
ㄴ HTTP 요청, HTTP 응답
비즈니스 계층 :  Service
ㄴ 비즈니스 로직을 실행
영속계층 : Repository
ㄴ 데이터베이스와 관련된 처리(CRUD)

DTO : 계층간 이동할 때 데이터를 담는 객체

ORM(Object Relation Mapping) : Java객체와 관계형 데이터베이스 테이블을 연결해주는 기술

orm이 없으면 JDBC를 이용해서 코드를 직접 다 작성해야 한다.
ORM을 사용하면 SQL보다는 객체 중심으로 데이터베이스를 다룰 수 있다.

Entity
테이블과 1대1로 매칭되는 클래스
@Entity를 붙여야 Entity로 취급을 한다.
- Entity클래스로 만든 객체 -> 테이블의 1개의 행에 해당한다.

SQL Mapper
- SQL을 개발자가 직접 작성하는 프레임워크
MyBatis
- 개발자가 SQL을 직접 작성하고, 그 SQL의 실행 결과를 Java객체에 연결해주는 역할을 한다.

복잡한 SQL 제어
ORM : 상대적으로 불편할 수 있다
SQL MAPPER : 직접 SQL을 작성하기 때문에 상대적으로 자유로움

Todo 추가하기
id : 자동생성
userId : 임시 아이디
title : 보내줘야함
done : 기본값 false

실제 서비스는 많은 사람들이 로그인을 해서 사용하기 때문에 인증 매커니즘을 구현을 해야 한다.

인증
- 사용자가 누구인지 확인하는 과정
- 사용자 아이디와 비밀번호
- 다중 요소 인증(sms, email)
- 생체인증(지문, 안면 인식)
- 소셜 로그인(페이스북, 구글, 네이버, 카카오)

인가
- 사용자가 인증된 후에, 그 사용자가 무엇을 할 수 있는지 결정하는 과정
- 특정 자원이나 기능에 접근할 수 있는 권한을 부여하는 절차

Basic 인증
- Todo프로그램은 로그인을 제외하면 상태를 유지해야할 이유가 없다.
- 상태가 없는 웹 어플리케이션에서 인증을 구현하는 방법은 모든 HTTP요청에 아이디와 비밀번호를 같이 보내는것이다.
- 아이디와 비밀번호를 Base64 인코딩하여 HTTP 요청 헤더에 포함한 뒤 요청을 보낸다.
- 서버는 데이터를 받아서 디코딩하여 사용자가 인증할 수 있는지 확인하고, 자격이 유효하면 요청한 자원에 접근을 허용한다.

토큰 기반 인증
사용자가 인증에 성공한 후 토큰을 발급받아, 이를 사용해 서버와의 통신에서 인증을 처리하는 방식
- 이 방식은 세션 기반 인증과 달리, 서버에 사용자의 상태를 저장하지 않고, 토큰을 통해 인증을 관리하기 때문에 무상태 방식으로 작동한다.
- RESTful API와 같은 분산 시스템에서 많이 사용된다.

토큰
- 사용자를 구별할 수 있는 문자열
- 서버가 자기만의 규칙으로 토큰을 만들어준다.

사용자가 로그인
- id랑 비밀번호를 전달

서버가 사용자 인증
- id랑 비밀번호를 확인하여 사용자가 올바른 자격을 가지고 있는지 확인

토큰 발급
- 인증에 성공하면, 서버는 사용자의 정보 또는 권한을 포함한 토큰을 생성하여 클라이언트게 반환한다.
- 이 토큰은 주로 JWT(JSON Web Token)형식으로 사용한다.

클라이언트가 토큰을 저장한다
- 브라우저에 저장을 한다.

토큰을 포함해 요청을 보낸다.
- 클라이언트는 서버에 요청을 보낼때마다 이 토큰을 HTTP 요청 헤더에 포함시켜 인증을 처리한다.

서버가 토큰을 검증
- 서버는 클라이언트로부터 전달받은 토큰이 유효한지 확인한다.
- 토큰이 유효하다면, 서버는 요청을 처리하고 필요한 자원에 접근을 허용

응답 반환
- 서버는 클라이언트에게 요청된 데이터를 반환하거나, 필요한 처리를 수행한다.

JWT
- JSON 형식으로 사용자 정보를 저장하고 서명을 통해 토큰의 무결성을 검증하는 토큰 기반 인증 방식

JWT는 세 부분으로 나뉜다.
1. 헤더
2. 페이로드
3. 서명
이 세 부분을 Base64 URL 인코딩으로 인코딩 한 후 마침표로 구분해 하나의 문자열로 합쳐진 것이 JWT이다.
