화면 설계
UI 요구사항 확인
UI에 대한 정의 -> 화면으로 보는 모든것 색상, 폰트, 버튼, 요소들의 배치
UI 구성요소
1. 시각적 디자인 : 색상, 글꼴, 아이콘, 이미지, 레이아웃
2. 상호작용 디자인 :시스템의 반응
3. 정보구조 : 사용자가 원하는 정보를 쉽게 찾을 수 있도록 만드는 설계
ㄴ 메뉴 : 기능을 어떻게 나눌지 -> 홈, 검색, 장바구니, 마이페이지
ㄴ 콘텐츠 분류 : 정보를 어떤 기준으로 묶을지 -> 최신순, 인기순, 카테고리별
ㄴ 화면 흐름 : 사용자가 어떤 순서로 화면을 이동할지 -> 목록 -> 상세 -> 결제
UX에 대한 정의 -> 사용자가 서비스를 사용하면서 느끼는 전체적인 경험
UX 구성요소
1. 사용성 : 별도의 설명 없이 얼마나 쉽게 이용할 수 있냐
ㄴ 버튼이 큼, 메뉴의 위치가 직관적임
2. 유용성 : 사용자가 요구한 기능이 있는가?
ㄴ 서비스에 사용자가 요구한 기능이 있는지
3. 매력도 : 서비스가 사용자에게 얼마나 매력적인지 정도
ㄴ 아이콘, 폰트 등이 마음에 들어야 사용을 한다.
4. 접근성 : 연령, 장애 여부, 환경에 상관없이 사용할 수 있는가
ㄴ 장애인, 고령자도 서비스를 이용할 수 있는가
5. 신뢰성 : 서비스에 대한 사용자의 믿는 정도
ㄴ 광고가 많다, 오류가 자주 발생한다 -> 신뢰도가 낮다.

UI/UX의 관계
- 다른 개념이지만, 실제 서비스에서는 떼놓을 수 없는 관계
UI가 UX를 만드는 중요한 수단이다.
- 사용자는 서비스 내부 구조를 직접 보는게 아니라 대부분 화면, 버튼, 메뉴, 입력창, 색상, 문구를 통해 서비스를 경험하기 때문이다.

좋은 UI가 항상 좋은 UX를 보장하지는 않는다.
- 화면이 예쁘고 세련되어도 사용하기 불편하면 좋은 UX가 아니다.

좋은 UX를 위해서는 UI뿐만 아니라 전체 흐름도 중요하다

UI/UX 표준
- 화면을 설계할 때 지켜야 하는 공통 기준
- 사용자가 웹사이트나 앱을 일관성 있게, 편리하게, 안전하게, 접근 가능하게 사용할 수 있도록 정해둔 설계 원칙

ISO 9241-210
ISO : 국제 표준화 기구(산업 전반에 걸진 표준을 제정하는 곳)
적용 대상 : 사용자와 컴퓨터가 상호작용하는 대상 전부
주요 내용 : 사용자 중심 설계 6대 원칙
1. 사용자는 설계 과정 전반에 적극적으로 참여해야 한다.
- 실제 사용자가 요구사항 수집, 분석 등 전 과정에 참여하여 사용자 의견이 설계에 반영되어야 한다.
2. 사용자 중심 설계는 전체 사용자 경험을 고려해야 한다.
- 단순 기능 만족이 아니라 정서/인지적 경험을 포함해야 한다.
3. 설계는 반복적으로 이루어져야 한다.
- 의견을 피드백하고 보완하는 과정을 반복해야 된다.
4. 설계는 사용자와 소프트웨어 구축 목표 및 환경을 기반으로 한다.
- 사용자의 업무 환경을 고려해야 한다.
ㄴ 병원 : 데스크톱, 배달 앱 : 스마트폰
5. 사용자 중심 설계 팀은 여러 학문 분야가 함께 협력해야 한다.
- 좋은 UX는 한 사람이 만들 수 없다.
- 기획자, UI디자이너, UX 디자이너, 개발자, 관리자가 협업해야 한다.
6. 사용자 평가에 반드시 포함되어야 한다.
- 개발자끼리 테스트하는 것이 아니라 실제 사용자가 테스트 해야된다.

- 국내 지침

- 국제 UI/UX 표준

- 플랫폼별 디자인 가이드라인을 수집하고 분석

- 기존 유사 응용 소프트웨어 UI 설계 사례를 조사하고 분석

※ 목표 응용소프트웨어 : 앞으로 만들거나 개선하려는 대상 시스템
2. 목표 응용소프트웨어의 UI요구사항 확인하기
- 목표 응용소프트웨어의 기능 및 사용자를 분석

- 업무의 특성을 고려하여 UI 구성 요소 및 흐름을 파악한다.

- 사용자 면담 및 UI/UX 요구사항 수집하기

- UI 요구사항 도출 및 정리

- UI 요구사항 검토 및 확정

- 여러 화면에서 반복적으로 사용하는 UI 요소를 하나로 만들어놓은 재사용 가능 객체

UI 프로토타입 제작, 검토하기

UI 요구사항을 바탕으로 구성 요소를 정의하고 흐름을 설계하여 프로토타입 제작하기

1. UI구성 요소를 정의
- 사용자 요구사항 및 시나리오 분석 결과 확인
- 화면별 주요 기능을 식별
- 구성 요소 유형을 분류
- 각 구성 요소의 속성 및 역할 정의

2. UI 흐름을 설계한다.
- 주요 화면 목록 도출하기
화면 목록 정의 
화면 ID	화면명	주요 기능		비고
- 사용자 동선의 흐름 시나리오 작성하기
ㄴ 사용자 페르소나 설정하기
ㄴ 시스템 사용 목적 중심의 시나리오 작성하기
- 화면 동작 및 이동 조건 정의하기
ㄴ 각 화면의 실행 사전 조건이나 다음 화면으로 전환되는 조건 정의, 사용자의 동작에 따른 화면의 동작 및 이동 흐름을 정의한다.

4. UI 흐름도 작성하기
- 도출된 화면과 사용자 흐름을 시각적으로 정리하여 전체 화면 흐름 구조를 한눈에 볼 수 있도록 UI 흐름도를 작성한다.
ㄴ 주요 화면과 화면 간 연결 흐름을 노드로 구성한다.
ㄴ 화면의 조건 분기/예외 흐름을 포함하여 작성
ㄴ 반복 가능성, 되돌아가기 흐름도를 고려하여 보완한다.
- 예외 흐름 및 오류 처리 흐름을 설계한다.
ㄴ 일반 흐름에서 벗어나는 예외 상황에 대한 UI 처리 기능을 정의하고, 메시지/알림 등을 제공하는 방법과 문구를 설계한다.

3. 기능 중심의 간단한 UI 설계 초안을 제작한다.
- 기능 중심의 간단한 UI 레이아웃을 작성한다.
ㄴ 메인, 팝업과 같은 화면의 종류를 정의한다.
ㄴ 기능 흐름에 따른 화면 구성을 고안한다.
ㄴ 간단한 형태의 각 화면을 스케치한다.
- UI별 구성 요소 위치를 배치한다.
- 정보 구조 및 데이터 우선순위를 설정한다.
ㄴ 사용자가 정보를 효율적으로 이해할 수 있도록 중요한 정보와 기능을 어떤 순서로 보여줄지 정하고, 그에 맞는 정보 구조를 설계한다.
- 상호작용 과정 설명을 추가한다.
ㄴ 사용자가 시스템 사용 시 어떤 동작을 하는지 설명을 추가한다.
- 팀 내 공유하고 의견 수렴하기
- 작성한 화면 초안을 팀원들과 공유하고 기능/디자인/사용성 관점에서 의견을 수렴하여 반영한다.

4. 시각적으로 완성도 있고 동작 가능한 프로토타입 제작하기
- 색상, 폰트, 여백 등 시각 요소를 소프트웨어에 적용한다
ㄴ UI에 적용할 색상, 폰트, 여백, 아이콘 스타일 등 디자인 요소의 기준을 수립한다.
ㄴ 버튼, 입력창, 목록 형태 등의 공통 UI 요소에 스타일 적용하기
ㄴ 디자인 도구를 활용하여 실제 화면에 스타일을 적용한다.
- 실제 서비스처럼 보이는, 동작하는 프로토타입 UI를 제작한다.
ㄴ HTML, CSS를 이용하여 기능의 일부만 동작하는 프로토타입
ㄴ 사용 흐름에 따라 기능 및 버튼 위치를 정의하여 구현한다.
ㄴ 로그인 실패 경고, 입력 정보 오류 등과 같은 상태를 구현한다.
- 화면 간 링크 연결 및 상호작용을 구현한다.
ㄴ 화면 간 이동 흐름을 설계하기 위해 버튼이나 메뉴를 클릭했을 때 연결될 다음 화면을 구체적으로 정의
ㄴ 사용자가 조작하였을 때 어떤 반응이 일어나는지 추가한다.
ㄴ UI 설계 및 구현 도구를 활용해 실제로 동작하는 링크를 설정한다.
- 사용자와 개발 팀의 의견을 반영해 수정하고 반복 테스트를 수행한다
ㄴ 사용성 테스트를 통해 디자인에 대한 의견 수집
ㄴ 개선해야 할 항목을 기능 중요도와 구현 난이도에 따라 분류
ㄴ 수집한 의견을 반영해 프로토타입을 수정한다.
- 최종 프로토타입을 확정하고 공유할 수 있는 링크와 파일을 준비한다.
ㄴ 모든 화면, 링크, 오류 메시지가 포함되어있는지 최종 디자인을 검토
ㄴ 외부 공유할 수 있는 링크 생성
ㄴ 프로토타입과 함께 기능 요약 문서와 화면 구성을 작성하여 개발팀에 전달
