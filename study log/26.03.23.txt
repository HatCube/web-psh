git (버전관리 프로그램)

버전을 만들어서 관리하는 과정
작업 공간(작업) > 스테이지 (임시저장) > 커밋(버전 확정)

명령어로 관리하는 방법
1.변경된 파일을 스테이지로 올리는 법 : git add .
2.버전을 확정하는 방법 : git commit -m "메세지"


프로그램으로 관리하는 방법
> githubdesktop

1.작업공간에서 작업을 하면 자동으로 스테이지에 올라간다.
2.메세지를 적고
3.push origin 버튼을 눌러야 github로 올라감

java의 특징
운영체제(하드웨어와 소프트웨어의 중간 역할을 하는 프로그램)
1.운영체제에 구애받지 않는다,
 - 한번만 작성하면 어디서든지 실행이 된다.

2. 객체지향 프로그래밍(OOP)
> "O"bject "O"riented "P"rogramming

3.자동 메모리 관리메모리 > 컴퓨터 RAM 부품
프로그램을 실행하면 메모리의 일정량을 차지한다.

프로그램 내부에서 사용되지 않는 것들을 JAVA가 알아서 제거

자료형 : 데이터의 재질과 크기
