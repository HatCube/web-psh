OSI 7 계층
컴퓨터끼리 통신할 때 어떤 일을 단계별로 나눠서 처리하는 구조

데이터를 한번에 막 보내는것이 아니라 역할을 7단계로 나눠서 보내고 받도록 정리한 개념

OSI 7계층이 필요한 이유
예전에는 컴퓨터 제조사나 시스템 마다 통신 방식이 제각각이었다.
서로 통신이 어려웠다.

국제 표준화기구(ISO)가 통신 과정을 단계별로 나눠서 표준처럼 이해하자

1.통신 과정을 쉽게 이해할 수 있다.
>복잡한 통신을 한번에 보면 어렵다.
>기능을 층별로 나눠서 본다.

2.문제가 생겼을 때 어디서 고장났는지 찾기 위해서
ex) 인터넷이 안될 때
>케이블 문제인지, IP문제인지, 프로그램 문제인지 구별해서 볼 수 있게 된다.

3.장비와 기술의 역할을 정리하기 위하여
>허브,스위치,라우터,IP,TCP,브라우저,웹서버와 같은 것들이 어느 계층에서 동작하는지 구분할 수 있다.

각계층

1계층 : 물리계층(Physical Layer)
>데이터를 실제로 보내는 계층
>전기 신호, 빛, 무선 신호처럼 눈에 보이지 않는 실제 신호로 데이터를 전달한다.
>랜선, 광케이블, 허브, 리피터, 무선전파, 커넥터

허브(여러 컴퓨터를 연결하는 장치)
리피터 : 약해진 네트워크 신호를 다시 강하게 만들어주는 장치

이 계층의 특징
>데이터의 의미는 모른다.
>그냥 신호를 보낸다.
>문자인지 이미지인지는 식별 불가
>오직 비트(0,1) 수준으로 처리

택배를 실은 트럭

데이터 링크 계층
같은 네트워크 안에서 데이터가 제대로 가도록 처리하는 계층
MAC 주소를 사용함

MAC주소란 
네트워크 카드에 부여된 고유한 물리 주소이다.
장비 자체의 주소라고 보면 된다.

내PC의 랜카드 주소
스위치가 장비를 구분할 때 사용하는 주소

이 계층의 주요 역할
>프레임 단위로 데이터 구성
>같은 LAN안에서 목적지 찾기
>오류 확인
MAC 주소 기반 전달

대표 장비
스위치

스위치 : 여러장치를 연결하면서 필요한 장치에게만 데이터를 전달하는 네트워크 장비
목적지 장치에만 전송
충돌 거의 없음
속도 빠름

허브는 모든 장치에게 데이터를 전송(비효율)
충돌 발생 많다
보안 취약 

택배상자에 "이 건물(스위치)안에 누구에게 전달할지"를 적는 것

3계층 : 네트워크 계층 (Network Layer)
>서로 다른 네트워크를 넘어 목적지 까지 가는 길을 찾는 계층
여기서 가장 중요한 것이 IP 주소

IP주소
인터넷이나 네트워크에서 장치를 식별하기 위한 논리적 주소.

이 계층의 핵심 역할
>목적지의 IP주소 확인
>어느 경로로 갈지 결정
>여러 네트워크를 거쳐 전달

대표장비
>라우터
서로 다른 네트워크를 연결하고 데이터를 목적지까지 가장 적절한 경로로 보내는 장치

2계층이 "같은 건물 안에서 누구에게 줄까"라면, 3계층은 "서울에서 부산까지 어떤 길로 갈까"를 결정하는 단계

LAN(Local Area Network)
좁은 범위의 네트워크
>집
>학교
>회사 내부

MAN(Metropolitan Area Network)
도시 단위 네트워크
>하나의 도시 전체 연결

WAN(Wide Area Network)
넓은 범위의 네트워크
>국가
>전 세계

4계층 : 전송 계층(Transport Layer)
>데이터를 정확하고 안정적으로 전달할지, 아니면 빠르게 보낼지 결정하는 계층

TCP
UDP

이 계층의 핵심 역할
>데이터 분할
>순서 맞추기
>누락확인
>재전송
>흐름제어
>포트번호 사용
