UX : UI를 활용하여 운전자가 주행하면서 느끼는 승차감, 조작의 편리함, 직관성, 만족감, 피로도 등 '총체적인 경험과 지각'을 의미한다.

UI 상세 설계

UI 시나리오 작성 원칙
- UI 상세 설계에 있어서 시나리오 작성은 반드시 필요한 사항이다.
- 화면 구성뿐만 아니라, 사용 흐름, 상호작용, 오류 처리, 접근성, 적용 수준까지 함께 설계 해야한다.

1. 사용자 상호작용 중심 설계
사용자 상호작용 : 사용자가 목표를 달성하기 위해 시스템과 주고받는 행동과 반응의 흐름
- 시나리오의 출발점은 항상 "사용자 행동"이어야 한다.
- 핵심 기능과 서비스별로 사용자의 상호작용 흐름을 정의하고 주요 단계를 시나리오 형태로 설계한다.

2. 적용 수준 구분
- 서비스에 적용할 기능이나 UI 요소를 필수, 권장, 우수로 나누어 정리하는 것
- 한 번에 다 개발할 수 없기 때문에 반드시 구현해야 하는 기능, 추후 개선할 기능을 구분해야 한다.
필수 : 서비스 제공을 위해 반드시 필요한 수준
권장 : 사용사 편의성을 높이기 위해 적용하는 수준
우수 : 더 나은 사용자 경험을 제공하기 위한 고도화 수준

3. 접근성과 포용성 반영
- 각 시나리오 단계에는 접근성 기준을 적용하며, 고령자와 장애인을 포함한 다양한 사용자 환경을 고려하여 설계한다.
라벨 제공 : 어떤 정보를 입력해야 하는지 명확히 표시
키보드 제어 : 마우스 없이 Tab, Enter키로 조작할 수 있게 하는 것
색상 대비 : 배경과 글자를 잘 구분할 수 있게 하는 것
오류 안내 : 오류 원인과 해결방법을 함께 제공한다.
대체 텍스트 : 아미지나 아이콘에 설명을 같이 제공을 하는 것

4. 상호작용, 피드백, 오류 처리
피드백 : 사용자가 화면에서 행동을 했을 때, 그에 대한 반응
-  사용자의 행동 이후 시스템의 반응, 입력 오류 처리, 장애 발생 시 대응 절차를 시나리오에 포함한다.

피드백 예시
버튼 클릭 : 메시지 표시
필수 항목 누락 : 누락된 항목 위치로 이동시키고 오류 메시지를 표시
서버 오류 : "오류가 발생했습니다, 잠시 후 다시 시도해주세요" 안내

오류 처리의 예시
필수 항목에 대한 미입력 -> xxx를입력해주세요 -> 해당 입력창에 포커스 이동
로그인 실패 -> 아이디나 비밀번호가 올바르지 않습니다. -> 로그인 화면을 유지

5. 공통 요소와 반복 패턴 활용
- 시나리오는 가이드라인에서 제시한 컴포넌트, 기본 패턴, 스타일 가이드를 기반으로 작성하고 재사용이 가능하도록 구조화한다.

컴포넌트 : ui를 구성하는 재사용 가능한 부품(버튼, 입력창, 메뉴)
기본패턴 : 사용자가 자주 경험하는 화면 흐름
스타일 가이드 : 버튼, 색상, 폰트 ,용어, 글자크기 등을 어떻게 사용할지 정한 규칙

6. 실제 사용자 데이터 기반 구성
