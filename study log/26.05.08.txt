auto_increment
>숫자 값을 자동으로 1씩 증가시켜서 입력해주는 속성
>데이터를 추가할 때마다 직접 번호를 입력하면 번거롭기 때문에 사용을 한다.
>반드시 숫자 타입의 컬럼에만 붙여야한다.
>인덱스가 있어야 하기 때문에 PK와 함께 사용한다.
>한 테이블에 하나만 사용할 수 있다.
> 직접 값을 넣을 수도 있다.
insert into board(board_id,title,content)
values(100,'관리자 공지','직접 번호 입력');
그 다음자동 증가 값은 보통 101부터 시작한다.
> 중간번호는 재사용하지 않는다.
1. 첫번째글
2.두번째글
3.세번째글 

delete from board
where board_id = 3;

새 글을 추가하게 되면 board_id가 4로 추가된다.
>auto_increment는 연속 번호 보장용이 아니라 고유 식별자 생성용이다.
>insert 했다가 rollback해도 다음번호로 건너뛴다.
>테이블 생성 후 다음 자동 증가값을 바꿀수 있다.
alter table 테이블명 auto_increment = 1000;
이후 새 데이터는 1000부터 들어갈 수 있다.

>현재 auto_increment 값 확인하기
show table status like '테이블명';

마지막으로 생성된 번호 확인하기
select last_insert_id();

stored program의 핵심 구조
delimiter //
create procedure hello()
begin
	select'hello MySQL stored program' as message;
end //
delimiter ;


※delimiter
MySQL에서 기본적으로 세미콜론을 만나면 SQL문장이 끝났다고 판단한다
프로시저 전체가 끝날때까지 ;로 끝난것으로 보지 말고 // 를 만났을때 끝난것으로 판단해라

begin ~ end는 여러 SQL문장을 하나의 불록으로 묶는 문법
begin
	문장1;

	문장3;
end

변수를 선언하는 declare
declare 변수명 자료형 default 기본값;

declare v_count int default 0
declare v_name varchar(50) default '홍길동'

begin ~ end 블록의 가장 앞부분에 작성해 한다.
일반 SQL문 뒤에 선언하면 오류가 난다.

begin
declare v_num int default 0;

select '시작';
end;

변수에 값을 넣는 set
set 변수명 = 값;

begin
declare v_num int default 0;
set v_num = 100;

select '시작';
end;

매개변수 in,out,inout
in은 외부에서 값을 받아오는 매개변수
create procedure 프로시저명 (in 매개변수명 타입)

out은 프로시저 내부에서 만든 값을 외부로 내보내는 매개변수
create procedure 프로시저명 (out 매개변수명 타입)

inout은 값을 받아오고 수정한뒤 다시 내보내는 역할
create procedure 프로시저명(inout 매개변수명 타입)
