집계 함수(그룹 함수)
- 하나이상의 행에 대한 값들에 대해 집계(개수,총합,평균,최대,최소)하여 1개의 값으로 반환
- 일반적으로는 일반 속성과 집계함수를 SELECT 절에서 같이 사용할 수는 없다.
- NULL값은 집계하지 않는다.

select avg(salary) from employees;

group by
- 같은 데이터를 묶어서 집계할 때 사용하는 문법
- 부서별
- 상품별
- 카테고리별

select department_id, avg(salary)
from employees
group by department_id;

그룹을 지을 때 꼭 한가지 속성만 쓸 수 있는것은 아님
-- 여러가지 속성으로 그룹지을 수 있다.
select department_id, job_id, avg(salary)
from employees
group by department_id,job_id;

having
- group by로 만든 그룹을 조건으로 필터링할 때 사용하는 문법
- where은 -> 전체 행에 대해서 필터링
- having -> 그룹핑을 하고난 다음 필터링
- group by절 뒤에 적어야 한다.

SELECT
FROM
WHERE
GROUP BY
HAVING

정렬 (order by)
>조회 결과를 원하는 순서로 정렬할 때 사용하는 문법
>오름차순(ASC) : 작은거 > 큰거 
>내림차순(DESC) : 큰거 > 작은거

select 절의 맨 뒤에 적는다.

select 컬럼명
from 테이블 명
order by 정렬기준컬럼 정렬방식

오름차순
숫자 : 작은값 > 큰값
문자 : ㄱ > ㅎ, A > Z
날짜 : 오래된 날짜 > 최신 날짜

여러개의 컬럼으로 정렬할 수 있다
order by 정렬컬럼1 정렬방식, 정렬컬럼2 정렬방식...;

순위를 정하는 함수(윈도우 함수)
ROW_NUMBER() : 무조건 1,2,3,4 처럼 번호를 붙힌다.
RANK() : 공동순위를 인정하고, 다음 순위를 건너뛴다.
DENSE_RANK() : 공동 순위를 인정하지만, 다음 순위를 건너뛰지 않는다.
NTIKE(n) : 데이터를 n개의 그룹으로 나눈다.
PERCENT_RANK() : 상대적인 순위 비율을 구한다.

순위함수() OVER(ORDER BY 정렬기준 컬럼 정렬기준)

index
테이블에서 데이터를 더 빠르게 찾기 위해 만드는 검색용 목차

인덱스가 없는 경우
id	이름		나이
1	김철수	20
2	이영희	22
3	박민수	21
4	최유리	23

select * from student where name = '박민수';
인덱스가 없으면 mysql은 데이터를 처음부터 하나씩 확인한다
이런 방식을 full table scan(전체 테이블 스캔)이라고 한다.

인덱스 생성 기본 문법 
create index 인덱스명
on 테이블명(컬럼명)

create index idx_student_name
on student(name);

name 값을 기준으로 정렬된 별도의 구조를 만든다.
[김철수] > 데이터 위치 
[박민수] > 데이터 위치
[이영희] > 데이터 위치

실제 테이블을 이름순으로 바꾸는게 아니라 이름순으로 찾기 좋은 별도의 목차를 만든다.

학교에 전교생이 3천명이 있음 > 홍길동 
반별로 출석부를 봐 > 홍길동

쿼리를 실행하면 인덱스가 자동으로 사용된다.

인덱스 확인하는 방법
show index from 테이블명;

인덱스 삭제하는 방법
drop index 인덱스명 on 테이블명;

테이블 만들때 pk를 설정하면 mysql은 자동으로 인덱스를 생성한다.
그래서 pk를 이용해서 select를 할때는 빠르게 동작할 가능성이 높다.

unique 제약 조건도 자동으로 인덱스를 만든다.

인덱스의 장점
>select 속도가 향상된다.
검색 조건에 index가 있으면 조회 속도가 빨라질 수 있다.

order by 성능 향상
>정렬을 빠르게 할 수 있다.

인덱스의 단점
1.저장공간이 추가로 필요하다.
>인덱스는 별도의 자료구조 이기 때문에 테이블 데이터를 저장하는 곳 외의 인덱스 공간이 추가로 필요하다.

2.insert가 느려질 수 있다.
>데이터를 추가할 때 테이블에만 넣는것이 아니라 인덱스도 같이 갱신해야 한다.

3.update가 느려질 수 있다.
>인덱스가 걸린 컬럼을 수정하면 인덱스도 다시 정리해야 한다.

4.delete가 느려질 수 있다.
데이터를 삭제할 때도 인덱스에서 해당 값을 정리해야 한다.

인덱스를 걸면 좋은 컬럼
1.where 조건에 자주 사용되는 컬럼
2.join에 자주 사용되는 컬럼
3.order by에 자주 사용되는 컬럼
4.group by에 자주 사용되는 컬럼

인덱스를 걸면 애매한 컬럼
>값의 종류가 너무 적은 컬럼
