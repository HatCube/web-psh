git (버전관리 프로그램)

버전을 만들어서 관리하는 과정
작업 공간(작업) > 스테이지 (임시저장) > 커밋(버전 확정)

명령어로 관리하는 방법
1.변경된 파일을 스테이지로 올리는 법 : git add .
2.버전을 확정하는 방법 : git commit -m "메세지"


프로그램으로 관리하는 방법

1.작업공간에서 작업을 하면 자동으로 스테이지에 올라간다.
2.메세지를 적고
3.push origin 버튼을 눌러야 github로 올라감

변수 : 데이터를 담는 상자
프로그램에서 데이터를 저장하기 위해 메모리 공간에 이름을 부여한 것

자료형 : 데이터의 재질과 크기
변수에 저장할 데이터의 종류과 크기를 정의하는 것

종류
정수형
byte = 1byte = 8bit
short = 2byte
int = 4byte
long = 8byte

8bit = 0000 0000
비트중 가장 앞자리 ("0"000 0000) > 부호 결정용 > 1이면 음수, 0이면 양수

공부기록 남기기(꾸준히 기록을 남기기)
그날 이해가 잘 안되었던 것, 해결이 안되었던 것 >> 해결 과정을 기록

변수를 만드는 방법
1. 변수의 선언

자료형 변수명; > 값이 들어있지 않은 상태

2. 변수에 값을 대입
변수명 = 값;

대입은 원하는 만큼 가능.
대입을 하면 이전의 값은 사라진다.

3.변수의 초기화(initialize)
자료형 변수명 = 값;

초기화 후에 대입이 가능함
대입은 2번 처럼.
