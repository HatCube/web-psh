화면 설계
1.요구사항 확인하기
1-1.요구사항 확인
>UI개념, UX개념
>UI/UX의 요소, 관계
>UI/UX의 표준(ISO 9241-210)
수행하기 / UI 요구사항 확인하기
>소프트웨어 개발을 위한 사용자 화면 표준 및 지침 확인
>>국내 치침, 국제 표준, 플랫폼별 고려사항, 실현 가능성
>목표 응용소프트웨어의 UI 요구사항을 확인한다
>>기능 및 사용자 분석,UI 구성 요소 및 흐름 파악, 사용자 면담 및 UI/UX 요구사항 수집 
>>UI 요구사항 도출 및 정리 > UI 요구사항 명세서를 작성

*명세서 작성시 표준이 있는건 아님
ID, 요구사항 설명, 유형(기능 / 비기능), 우선순위, 난이도, 중요도

>UI 요구사항을 검토하고 확정 > 피드백 수정을 거치면서 버전을 기록


1-2.프로토타입 제작 검토
>프로토타입의 개념
>프로토타입의 유형(초기 Low-fidelity, 모사형 High-fidelity)
>프로토타입 제작 절차
>프로토타입 설계 및 제작 도구
>>초기 프로토타입 설계 도구(Adobe XD,Sketch,Balsamiq)
>>모사형 프로토타입 설계 도구(Figma,UXPin,ProtoPie)

수행 내용 / UI 프로토타입 제작, 검토하기
>UI 요구사항을 바탕으로 구성 요소를 정의하고 흐름을 설계하여 프로토타입 제작
>>UI구성 요소 정의
>>UI 흐름 설계
>>기능 중심의 UI 설계 초안 작성
>>모사형 프로토타입 제작
>제작한 UI 프로토타입의 적정성을 검토한다.
>>사전에 정의한 스타일 가이드와 디자인 기준을 준수하였는지 확인
>>버튼,입력창 등에 동일한 기능 요소가 일관적으로 적용되었는지 확인
>>시각적 요소인 아이콘과 텍스트 정렬 상태, 오타 및 간격 점검
>>상호작용 시 상태 표현(활성/비활성/에러등)의 일관성 확인
>유스케이스 흐름에 따른 화면 전환 및 기능 배치 여부 확인
>오류 발생 시 경고창이나 안내 매시지가 적절하게 처리 되었는지 점검

행위 순서 규정
>사용자의 행동 과정을 세분화하여 순서대로 명확하게 안내하되,필요할 경우 사용자가 의도에 맞게 순서르 자유롭게 바꿀 수 있다.
>불필요하게 많은 상호작용으로 인해 작업 진행이 방해받지 않도록 주의해야 한다.
>이를 위해 미리보기와 같은 기능을 제공하면 사용자가 예상 결과를 사전에 확인할 수 있어 대부분의 경우 만족돌르 높일 수 있다.

4.스토리보드 작성 방법
>스토리보드는 서비스 구축 과정에서 디자이너와 개발자가 공통으로 참고하는 최종 산출물이다.
>정책,프로세스,콘텐츠 구성,와이어프레임,기능정의, 데이터베이스 연동등 서비스 구현에 필요한 핵심 정보가 들어가 있다.
