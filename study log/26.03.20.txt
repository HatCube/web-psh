git congif --global user.(name,email) "" > name은 닉네임 email은 이메일로 커밋 작성시의 기록할 유저의 정보 변경

git init > 이 폴더에 위임

git add > 이 폴더에서 작업한 것들 임시저장(.은 변경점 전체, 파일명은 그 파일만 변경)

git commit > 이 폴더에서 작업한 것들을 저장 및 업로드

git log > 이 폴더에 커밋 한 것들 기록을 확인

git reset > 커밋을 취소
>>(--soft 는 임시저장 상태로 복귀
    (--mixed는 임시 저장도 취소하고 작업 공간에는 남겨둔다
    (--hard는 임시 저장도 취소하면서 작업 공간에서도 삭제
    (또한 HEAD는 최신버전을 의미)

github에 올리기 전에 반드시 커밋이 전부 완료 되어있는지 확인할것

git remote add origin (링크)> 깃허브와 로컬 저장소를 "origin"이라는 이름으로 연결

git push -u origin master > 로컬 저장소에 커밋 했던 것들을 깃허브 "마스터"에 업로드

add (임시저장) > commit(로컬 저장소에 업로드) > push (원격저장소에 저장)

자바 프로젝트 구성
src폴더
- 소스코드가 저장되는 폴더
- 패키지 단위로 폴더가 만들어지고, 그안에 파일이 위치한다.
- 작성하는 모든 파일은 이곳에 저장된다.

JRE System Library
 - 프로젝트 실행에 필요한 코드들의 묶음
 - 기본적인 기능이 들어있다.
 - 프로젝트 생성시 자동으로 추가된다.

패키지
 - 관련있는 클래스 끼리 묶어놓는 폴더 개념
 - src폴더 안에 하위 폴더로 생성된다.

클래스
 - 자바 프로그램의 기본 구성 단위이다.
 - 자바에서 모든 코드는 반드시 클래스 블록({})안에 있어야 한다.

메서드
 - 클래스 안에서 정의된 함수
 - 기능

클래스를 만든다.
메인 메서드를 만든다.
메인 메서드 영역 안에 코드를 작성한다.
