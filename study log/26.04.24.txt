미들웨어
프로그램과 운영체제 또는 여러 시스템 사이에서 서로 연결하고 도와주는 프로그램

미들웨어는 사용자가 직접 눈으로 보는 프로그램은 아니지만 시스템이 정상적으로 동작하기 위해 뒤에서 중요한 역할을 한다.

1.미들웨어를 사용하는 이유
1.서로 다른 시스템을 연결하기 위해
>시스템 마다 언어,데이터형식,통신방식이 다를 수 있다.
이러한 시스템들을 연결해 주는 역할

2.개발자가 공통기능을 직접 만들지 않아도 된다.
로그인 세션 관리,보안처리,데이터 변환, 메시지 전송
이런 기능을 개발자 전부 직접 만들면 시간이 오래 걸리고 실수도 많아진다.
미들웨어는 공통 기능을 제공해준다.

3.시스템을 안정적으로 운영하기 위해
짧은 시간에 요청이 몰리는것들을 처리
>요청을 여러 서버로 나눠서 보냄
>동시에 들어온 요청을 순서 있게 처리

3.데이터 베이스 관련 미들웨어
db서버를 따로 운영하기 때문에 db서버와 어플리케이션 사이의 연결을 관리.

db연결관리
SQL실행
조회결과를 객체로 반환
트랜잭션 처리

JDBC
DB Connection Pool
MyBatis
Hibernate
JPA

4.메시지 지향 미들웨어
>시스템끼리 메시지를 주고받게 해주는 미들웨어
메시지 : 시스템끼리 주고받는 데이터 묶음

RabbitMP
Apache Kafka

5.TP모니터 (트랜잭션 처리 미들워에)
>대량의 트랜잭션을 안정적으로 처리하는 미들웨어
트랜잭션 : 반드시 하나의 작업처럼 성공하거나 실패해야하는 처리단위 

Tuxedo
Tmax
CICS

주로 금융 공공기관 대형 기업 시스템에서 사용된다.

6.연계 통합 미들웨어 EAI / ESB

EAI : 허브 & 스포크
ESB : 메시지 버스(결합력이 낮다.)

로그인 했는가?

cpu사용량

Scouter

데이터 베이스 기초 활용하기
데이터(Data) : 값을 나타내는 것 숫자,문자,이미지,비디오,날짜 
데이터베이스(DataBase) : 데이터를 활용하기 위해 통합하여 저장한 운영 데이터의 집합

실시간 접근성
계속적 변화
동시 공용

데이터 베이스 관리 시스템(DBMS)
>사용자가 데이터베이스에 접근하여 쉽게 사용할 수 있도록 관리 해주는 시스템 

유형
계층형 - 데이터가 트리구조로 저장
네트워크형 - 그래프 구조
관계형 - 테이블 형태로 저장
객체지향형

스키마
>데이터베이스 안에서 데이터를 어떻게 저장할지 구조를 정의한 것

외부스키마
>사용자 또는 프로그램이 보는 데이터 구조
>사용자마다 다르게 보일 수 있음
>필요한 데이터만 보여줌
>보안에 중요함

개념스키마
>데이터베이스 전체 구조를 정의한 것
>모든 데이터의 구조를 정의
테이블,관계,제약조건 포함
>DB설계의 중심

내부스키마
>데이터가 실제로 저장되는 방식
>디스크에 어떻게 저장되는지 정의
>인덱스, 파일 구조 포함
>성능과 관련 있음

SQL을 작성하고 실행하게 되면 > 임시로 반영(결과 확인 가능) > 마음에 들면 COMMIT
>마음에 안들어 ROLLBACK (주의할점 > COMMIT하고 나서 롤백은 안됨)
