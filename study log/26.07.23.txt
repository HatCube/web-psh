공통모듈구현
자주사용하는 기능을 클래스나 메서드 단위로 만드는것
- 함수 단위
- 컴포넌트
- 어플리케이션 단위

응집도
- 모듈 내부에서 구성 요소 간 밀접한 관계를 맺고 있는 정도
- 응집도가 높을 수록 품질이 높다고 평가한다.

기능적 응집도
- 모듈 내부의 모든 기능이 단일한 목적을 위해 수행되는 경우
순차적 응집도
- 모듈 내에서 한 활동으로부터 나온 출력값을 다른 활동의 입력값으로 사용하는 경우
통신적 응집도
- 동일한 입력과 출력을 사용하여 다른 기능을 수행하는 활동들이 모여 있을 경우
절차적 응집도
- 모듈이 다수의 관련 기능을 가질 때 무듈 안의 구성 요소들이 그 기능을 순차적으로 수행할 경우
시간적 응집도
- 특정시간에 처리되어야 하는 활동들을 하나의 모듈에서 처리하는 경우
논리적 응집도
- 유사한 성격을 갖거나 특정 형태로 분류되는 처리 요소들이 한 모듈내에서 처리되는 경우
우연적 응집도
- 모듈 내부의 각 구성요소들이 연관이 아예 없을 경우

결합도
- 모듈과 모듈간의 관련성 정도를 나타내며, 관련이 적을수록 모듈의 독립성이 높아져 모듈 간 영향이 적어지게 된다.

자료 결합도
- 모듈 간의 인터페이스로 전달되는 파라미터를 통해서만 모듈 간의 상호작용이 일어나는 경우이다.
스탬프 결합도
- 모듈 간의 인터페이스로 배열, 객체, 구조 등이 전달되는 경우
제어 결합도
- 단순 처리할 대상인 값만 전달되는 게 아니라 어떻게 처리를 해야 한다는 제어 요소가 전달되는 경우
외부 결합도
- 다수의 모듈이 모듈 밖에서 도입된 데이터, 프로토콜, 인터페이스 등을 공유할 때 발생하는 경우
공통 결합도
- 파라미터가 아닌 모듈 밖에 선언되어 있는 전역 변수를 참조하고 전역 변수를 갱신하는 식으로 상호작용하는 경우
내용 결합도
- 다른 모듈 내부에 있는 변수나 기능을 다른 모듈에서 사용하는 경우

공통모듈을 구현하는 방법
1. 문서나 설계도를 통해서 만들어야할 공통 모듈을 확인한다.
- 공통 모듈 관리 대장을 확인한다.(설계 단계에서 작성된 구현해야 할 전체 시스템의 공통 모듈 산출물)

2. 산출물을 기반으로 공통 모듈을 구현한다.
- 공통 모듈을 구현하기 위한 I/O 오브젝트(VO/DTO)를 정의한다.
DTO(DataTransfer Object)
VO(Value Object)
- 공통 모듈을 구현하기 위한 SQL을 작성한다
MyBatis : Java 객체와 SQL을 연결해주는 SQL Mapper 프레임워크
Mapper : Java코드와 SQL을 연결해주는 역할
XML(eXtensible Markup Language) : 확장 가능한 마크업 언어
ㄴ 태그를 직접 만들 수 있다.
ㄴ 데이터의 표현이 목적이다.

Mapper Interface
- Java에서 호출할 메서드를 정의

Mapper XML
- 실제 SQL을 작성한다

3. 데이터 접근 객체(DAO : Data Access Object)를 구현한다.
- DAO에서는 SQL문을 구현한 XML 파일에서의 ID값을 호출하여 데이터를 조작한다.

4. 서비스 클래스를 구현한다
- 동일한 시스템 내에서 사용되는 결합도가 낮은 공통 모듈을 구현하기 위해 스탬프 결합도 수준의 서비스를 구현한다.

5. 컨트롤러 클래스를 구현한다.
- 시스템 내부 및 외부에서 사용할 수 있도록 REST API를 통한 공통 모듈을 구현한다.

mapper-xml 작성시 규칙

<mapper namespace="dao랑 연결"> </mapper> 안에 SQL을 작성해야 한다.

<select id="findAll" resultType="UserVO"> select * from users</select>
<insert id="save"> insert into users ...</insert>
<update id="update"> update users set ...</update>
<delete id="delete"> delete from users ...</delete>

id : XML의 SQL과 Java 메서드를 연결
resultType : SQL 조회 결과를 Java객체로 변환

#{} : Java에서 전달한 값을 SQL에서 사용하기 위한 형식문자
UserVO findById(@param("abc") Long id) ->  select * from users where id = #{abc}

객체를 파라미터로 전달
int insert(UserVO vo) -> insert into users values( #{name}, #{email}, #{age})
vo.getName()
vo.getEmail() 처럼 동작을한다.

각 CRUD를 작성할 때 DAO 메서드의 반환값
SELECT : VO, LIST<VO>
INSERT : int
UPDATE : int
DELETE : int

전체 도서 개수 조회
count

<if>
- 조건이 참일 때만 SQL문장을 추가하는 문법

<if test="name != null">
	AND name=#{name}
</if>

저자를 가지고 책을 조회

<where>
- 조건이 존재할 때 자동으로 where을 붙여주는 태그

<choose></choose> : 여러 조건 중 하나를 선택하는 전체 영역

<when></when> : 조건을 검사

<otherwise></otherwise> : 모든 <when>조건이 거짓일 때 실행
