제어 역전(Inversion of Control)
우리가 직접 객체를 만드는것이 아니라, 스프링이 객체를 만들고 필요한곳에 주입해주는것

1. 스프링이 객체로 만들도록 클래스에 @Component 어노테이션을 붙인다.

@Component
public class Engine{
	public void start(){ "부릉"};
}

@Component
public class Car{
	Engine engine;

	engine.start();
}

2. 프로젝트를 실행하면 필드로 사용되는 곳에 만들어 놨던 객체를 주입하여 메서드를 사용할 수 있도록 해준다.

의존성 주입
1. 필드에 직접 주입
2. setter의 매개변수자리에 주입
3. 생성자의 매개변수 자리에 주입

gradle
다양한 프로그래밍 언어를 지원하는 빌드 자동화 도구
의존성 관리, 테스트 실행, 배포, 패키징 등의 빌드 작업을 자동화하며, 유연성솨 확장성을 중시한다.

필요한 라이브러리가 있으면 build.gradle파일의 dependencies 부분에 링크를 붙여넣으면 스프링이 자동으로 다운받는다.

mvnrepository에서 링크를 복사해온다.

CORS(Cross-Origin Resource Sharing)
교차 출처 리소스 공유

출처(Origin)
- 프로토콜, 호스트주소, 포트번호까지 모두 합친 주소를 의미한다.
- 서버의 위치를 찾아가기 위해 필요한 가장 기본적인 것들을 합쳐놓은 것

동일 출처 정책(Same-Origin Policy)
- 브라우저는 기본적으로 보안을 위해 동일 출처 정책이라는 규칙을 따른다.
- 같은 출처에서 로드된 웹사이트만 서로 데이터를 주고받을 수 있다.
- 다른 출처에서 데이터를 요청하는 것을 제한하는 정책
- 이 정책은 보안을 위해 존재하는데, 악의적인 웹사이트가 사용자의 브라우저를 이용해 다른 출처에서 데이터를 가져오지 못하게 막는 역할을 한다.
