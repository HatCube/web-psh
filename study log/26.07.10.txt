1. 개발환경 구축하기
1-1. 개발 환경 준비
응용 소프트웨어 : 특정 사용자의 요구를 충족시키기 위해 컴퓨터 시스템에서 실행되는 프로그램
응용 소프트웨어의 특징

- 도메인 특화
- 프로그램 구성요소의 집합
- 운영환경과 밀접한 연관

- 모바일 응용 SW
- 임베디드 응용 SW
- 클라우드 응용 SW : 구글 스프레드시트, Office 365
- 대규모 엔터프라이즈 SW
- AI/데이터 기반 SW

개발도구 : 개발자가 프로그램을 쉽게 만들 수 있도록 도와주는 도구

1-2. 개발 환경 구축

클라이언트 환경
- 클라이언트 서버 시스템에서 제공하는 서비스를 활용하기 위해 사용자와의 인터페이스를 제공하는 하드웨어 보통 PC, 웹 브라우저, 모바일 디바이스에 해당한다.
서버환경
- 목적에 따라서 어플리케이션 서버, 데이터베이스 서버, 파일 서버, 개발 빌드 서버, 테스트 서버 등으로 나눌 수 있다.

개발 소프트웨어 환경
1. 운영체제
2. JVM
3. Web Server(HTML, CSS, 이미지 정적 데이터를 처리)
4. WAS (동적인 웹 서비스를 제공)
5. DBMS
6. 요구사항 관리도구(고객의 요구사항을 수집 분석 추척 협업을 지원)
7. 설계 모델링 도구(UML, ERD, API 설계 도구)
8. 구현 도구(이클립스, VSCODE같은 코드를 작성하는 프로그램의 선정)
9. 테스트 도구(개발된 코드의 정확성 검증)

형상관리
- 소스코드 뿐만 아니라 문서, 설계 산출물, 스크립트, 빌드 결과물 등 다양한 개발 산출물을 관리 대상으로 포함하여, 협업 기반 개발 환경에서 필수적인 핵심 프로레스로 자리잡고 있다.

형상관리의 필요성
- 소프트웨어 변경 사항을 정확히 추적, 통제
- 다수 개발자 협업 시, 충돌 방지 및 동시 작업 지원
- 개발 운영 환경 간 차이 최소화
- 고객 요구사항의 지속적인 변경 대응

스프링부트 환경 구축하기
STS(Spring Tool Suite)
Spring Framework와 Spring Boot 어플리케이션을 빠르고 쉽게 개발할 수 있도록 만든 통합 개발 환경이다.

기존에 사용하던 eclipse에 스프링부트 프로젝트를 import시켜 사용하는 방법

프레임워크
- 프로그램을 만들 때 필요한 기본 구조와 공통 기능을 미리 제공하는 개발 틀
- 개발자 프로그램을 처음부터 만들지 않고, 이미 준비된 구조 안에서 필요한 기능만 작성하도록 도와주는 도구

스프링의 핵심 개념
1. 제어 역전(Inversion of Control, IoC)
- 객체의 생성과 관리를 개발자가 아닌 프레임워크나 컨테이너가 대신 관리하도록 하는 것
- 전통적인 객체 지향 프로그래밍에서 객체 간의 의존성을 개발자가 직접 처리하는 반면, IoC는 이 흐름을 역전시켜 스프링 프레임워크와 같은 컨테이너가 객체 간의 의존성을 주입하고 관리한다.
- 어플리케이션의 유연성과 유지보수성을 높이는데 기여하며, 스프링 프레임워크에서 가장 핵심적인 개념 중 하나

IoC의 주요 개념
1. 의존성 주입(Dependency Injection , DI)
- 제어역전의 구체적인 구현 방법으로, 객체가 스스로 의존성을 관리하는 것이 아니라, 외부에서 필요한 의존성을 주입하는 방식이다.
- 스프링에서 가장 일반적으로 사용되는 방법
- 필드 주입, 생성자 주입, 세터 주입을 통해 의존성을 주입할 수 있다.
2. 제어 역전
- 객체의 생성과 생명주기를 스프링 컨테이너가 제어하게 된다.
3. 스프링 IoC 컨테이너
- 스프링에서는 ApplicationContext라는 컨테이너를 통해 구현된다.
- 이 컨테이너는 어플리케이션에 필요한 객체(bean)를 생성하고, 필요한 의존성을 주입해준다.
