데이터 모델링
>현실 세계의 업무 내용을 데이터베이스에 저장할 수 있도록 데이터 구조로 설계하는 과정

1.데이터 중복의 발생 가능성
2. 수정이 어렵다
3.삭제이상 > 필요없는 데이터를 지우려다 필요한 데이터까지 삭제하는 경우
삽입 이상 > 필요한 데이터를 추가하려다 필요없는 데이터까지 추가하는 경우

관계를 문장으로 처리
학생은 수간 신청을 한다.
수업은 수강신청을 받는다
강사는 수업을 담당한다.



1NF 속성 하나에 하나의 값만 들어어가야한다.

제 4 정규형
한 엔티티가 서로 독립적인 여러 값 집합을 가지면 분리

물리 모델링
>논리적 모델을 실제 DBMS에서 실행 가능한 SQL로 바꾸는 것
1.컬럼 타입 구체적으로 정하기
2.NULL 허용 여부 정하기
3.중복 불가 조건 정하기
4.기본값 정하기(default)
5.외래키 제약조건 정하기

데이터에 대해서 조회 수정 추가 삭제하는 명령어
select 컬럼1,컬럼2,... from 테이블;

모든내용을 조회하고 싶다.
select * from 테이블;
