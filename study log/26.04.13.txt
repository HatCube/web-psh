컬렉션 프레임워크
프레임워크 : 인터페이스나 클래스를 준비해놓고 사용하는것

인터페이스
>상수
>추상메서드

List 인터페이스
>순서가 있다. (index가 존재한다)
>중복이 가능하다.
>넣는대로 들어간다.
ArrayList : List 인터페이스를 구현한 대표적인 클래스
> 조회에는 유리하다, 추가/삭제에는 불리하다.
,LinkedList,...
Set

>순서가 없다(index가 없기 때문에)
>중복이 안된다.(중복된 내용을 추가하려고 해도 반영이 안됨)

HashSet : Set 인터페이스를 구현한 대표적인 클래스

반복자(Iterator)
Set같은 index가 없는 컬렉션에서 데이터를 하나씩 꺼내줄 수 있는 방법
반복자 객체는 컬렉션 객체로부터 iterator()메서드를 사용해 생성이 가능하다.
List<Integer> list = new ArrayList();

list.iterator();

hasNext() : 다음에 출력할 요소가 있으면 true/ 아니면 false
>결과가 t/f로 나온다는것은 조건식(if), 반복문의 조건에 들어갈 수 있다.

next() : 컬렉션에서 요소를 하나 반환

향상된 for문도 내부에서 반복자를 기반으로 동작을 한다.

람다식 문법
반환타입, 메서드명을 생략하고 매개변수와 몸체 사이에 화살표를 쓴다.

생략법
1.매개변수의 타입을 생략할 수 있다.
>매개변수가 두개 이상일때 타입을 둘다 생략해야 한다.

2.매개변수가 1개일때는 소괄호 생략 가능
(x) -> {return x;}
x -> {return x}
>2개 이상일때는 안됨

3. 반환형이 void면 중괄호 생략 가능
> 명령이 return 한줄이면 return과 중괄호 함께 생략 가능
x ->System.out.println(x);
x ->{return x;};
x -> x;
