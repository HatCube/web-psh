Class 클래스의 역할 
>클래스의 정보를 알아와야할 때
>생성자를 읽어서 객체를 만드는것(프로그램이 실행되는 시점)

핵심
컴파일 시점에 확인하는 것이 아니라 실행시점에서 확인하기 위함이다.

어노테이션
코드에 붙이는 표식같은 것
클래스, 필드, 메서드 등에 붙여서 코드에 대한 추가정보를 전달할 때 사용.

코드 자체의 실행문은 아니지만 이 코드가 어떤 의미인지,컴파일러가 무엇을 확인해야 하는지, 프레임워크가 어떻게 처리해야 하는지와 같은 정보를 알려주는 역할을 한다.

어노테이션을 왜 사용할까.

1.실수를 줄여준다.
2.코드의 의미를 더 분명하게 보여준다.
자바가 제공하는 어노테이션
@Override
@Dreprecated : 더이상 없뎃이니까 그만써라
@SupressWarnigs : 경고 보기 싫을때 사용

메타 어노테이션
@REtention : 언제까지 정보를 유지할거냐
@Target : 어디에 적용할 거냐

나만의 어노테이션
public @interface 어노테이션명{}

제네릭
타입을 미리 고정하지 않고 나중에 정할 수 있게 만드는 문법
어떤 타입이든 담을 수 있는 틀을 만든 뒤 사용할 때 타입을 정하는 방식

제네릭을 사용하게 되면 
꺼낼때 형 변환이 필요없고 잘못된 타입을 넣으면 컴파일 오류가 난다.
타입마다 변수를 만들어줄 필요가 없어서 재사용성이 크게 증가한다.
제네릭 타입변수에 들어갈 수 있는 것은 클래스만 가능하다.

제네릭 메서드
메서드에만 적용되는 제네릭 타입을 따로 두는것
제네릭 클래스가 아니어도 제네릭 메서드를 만들 수 있다.

중첩 제네릭
제네릭 타입 안에 또 다른 제네릭 타입이 들어간 구조
Box<Pair<String,Integer>>

복잡한 구조의 표현
Board<article<Integer,Integer>>

코드 가독성과 유지보수에 유리
class Box <T>{
	T item;
}

Box<List<String>> box = new Box<>();
