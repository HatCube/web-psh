요구사항 확인 

1.현행 시스템의 분석
>새 시스템을 만들거나, 기존 시스템을 개선하기 위해 현재 사용중인 업무 방식,프로그램,데이터,서버,네트워크,문제점을 조사하고 정리하는 활동

목적
>새로운 시스템을 정확하게 만들기 위해 현재 상황을 이해한다.

문제점
> 1. 기존 업무의 흐름을 모른다면 필요한 기능을 빠트릴 수 있다.
> 2. 기존 데이터 구조를 모른다면 데이터 이전 시 오류 발생
> 3. 기존 시스템 연동을 모른다면 외부 시스템과 연결 실패사용자 불편 사항을 모른다면 새 시스템을 만들어도 만족도가 낮다.
5.현재 원인을 모른다면 문제 해결이 아닌 단순 재개발이 된다.

인터페이스의 현황 파악
>시스템과 시스템이 데이터를 주고 받는 연결 지점을 의미 

2단계 : 혀행 시스테의 아키텍쳐 및 소프트웨어 구성 현황 파악
>현재 어떤 시스템이 어떤 구조로 만들어져 있고,어떤 소프트웨어를 사용하고 있는지 파악

아키텍처 현황 파악
>현재 시스템이 어떤 구조로 만들어져 있는지 조사하는 일

소프트웨어 구성 현황 파악
> 현재 시스템이 어떤 소프트웨어를 사용하고 있는지 조사하는 일 

2단계 산출물
>시스템 아키텍처 구성도
클라이언트부터 인프라 까지 전체 구성을 한눈에 파악할 수 있다.

>소프트웨어 구성 목록 
시스템을 구성하는 소프트웨어를 영역별로 정리한 목록

>소프트웨어 버전 현황 표
현재 사용 버전,최신버전,업데이트 상태를 한눈에 파악할 수 있는 표

>기술 스택 정리표
역할별 기술 스택을 목적,선택이유,대안까지 함께 정리한 표

3단계 : 하드웨어 및 네트워크 분석
>현재 시스템이 운영되는 물리적 또는 클라우드 인프라 환경을 파악

하드웨어 구성 현황 파악
서버ㅡ저장장치,백업,장비보안등의 정보를 정리하는 것
클라우드 환경이라면 EC2,RDS,S3 같은 자원

하드웨어 현황 파악
>서버,저장장치,백업장비,보안장비등의 정보를 정리하는 것
클라우드 환경이라면 EC2,RDS,S3같은 자원을 확인한다.

포트 port
> 외부와 데이터를 주고받는 통로이자 접속 지점

SSL (Secure Sockets Layer)인증서
>웹사이트와 사용자 브라우저 간 데이터를 암호화하여 개인정보 노출을 막는 디지털 보안 솔루션

데이터 암호화
>제 3자가 내용을 볼 수 없도록 보호

데이터 무결성
>데이터가 전송중에 변조되지 않도록 보호

신원 인증
>클라이언트가 접속하는 서버가 신뢰할 수 있는 서버인지 확인

캐시(cache)
> 자주 사용하는 데이터나 값을 미리 복사해놓는 임시 장소를 가리킨가.
>저장공간이 작고 비용이 비싼 대신 빠른 성능을 제공한다.

발열이 심하고 너무 비싸져 현실성이 없다.
